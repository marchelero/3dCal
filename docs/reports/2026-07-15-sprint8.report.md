## Decisiones tecnicas

1. **`WidgetsBinding.ensureInitialized()` antes de `SharedPreferences.getInstance()`**: el plugin necesita el binding inicializado. Sin esto, dart报错 de "ServicesBinding.defaultBinaryMessenger was accessed before the binding was initialized."

2. **`sharedPreferencesProvider` con `throw UnimplementedError` por default**: fuerza a que main.dart o tests provean una instancia. Es un fail-fast para evitar uso accidental de una instancia async no resuelta.

3. **Debounce 500ms en save**: el usuario tipa rapido, no queremos escribir a disco en cada keystroke. 500ms es suficiente para capturar la "ultima intencion" del usuario sin ser molesto.

4. **Restore en `addPostFrameCallback` (no en `initState`)**: el `load` es async, no se puede hacer sync en `initState`. Ademas, primero se auto-poblan los defaults (filamento/impresora) y DESPUES se override con el draft, asi el draft gana si existe.

5. **Material list NO se restaura**: la lista de materiales del modo advanced se serializa en JSON pero el calculator no la rehidrata (AnimatedList state es complejo). Si el usuario estaba en advanced con 2 materiales, al reabrir queda en express con los 8 inputs simples. Trade-off: simplicidad vs fidelidad. Mejora futura.

6. **Listeners agregados en `initState` + dispose cancela timer**: cada controller tiene un listener permanente. Si el widget se destruye antes del debounce, el timer se cancela (no se persiste estado obsoleto). El listener se libera cuando el controller se dispose (en `dispose()`).

7. **Shared widgets con `onRetry` opcional**: el `ErrorView` recibe un callback. Si se provee, muestra el boton "Reintentar" que invalida el provider. Si no, solo muestra el error. Patron consistente con `ref.invalidate(provider)`.

8. **`SharedPreferences.setMockInitialValues({})` en tests**: necesario para que el `getInstance()` no falle en el environment de Flutter test (no hay platform channel real).

## Issues encontrados y resueltos

1. **`sort_constructors_first` en `calculation_draft.dart`**: el lint queria `const constructor` + `factory constructor` ANTES de los fields. Reorden: const → factory → fields → methods.
2. **`use_build_context_synchronously` en calculator_page.dart linea 261/266**: el `ScaffoldMessenger.of(context)` se llamaba despues de un `await`. Solucion: agregar `if (!mounted) return;` despues del await.
3. **`directives_ordering` en dashboard_page.dart**: orden de imports por path ASCII. `../../../../` (4-deep) viene antes de `../../../` (3-deep) y `../` (1-deep). Reorden: shared/.../empty_view, error_view, loading_view, calculation/.../dashboard_stats, ./profit_bar_chart.
4. **`unnecessary_underscores` en filaments_page_test.dart**: `(_, __)` → `(_, _)`. Dart 3.7+ permite multiple `_` parameters.
5. **`DecimalInputField` usa `TextField` no `TextFormField`**: el test original usaba `find.widgetWithText(TextFormField, ...)`. Cambio a `TextField`.
6. **`UnimplementedError` en tests existentes**: calculator_page_test y sprint0_smoke_test no overridean `sharedPreferencesProvider`. Solucion: agregar `SharedPreferences.setMockInitialValues({})` + override en cada test.
7. **`find.text('Aun no cotizaste nada')` en dashboard_page_test**: el nuevo `EmptyView` combina titulo y mensaje en un solo `Text` widget. Cambio a `find.textContaining('Aun no cotizaste nada')`.
