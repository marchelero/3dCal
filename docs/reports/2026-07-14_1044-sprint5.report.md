## Decisiones tecnicas

### Mode preservation en "Reusar"

1 material guardado → modo `express` al reusar.
2+ materiales → `advanced`. Asi el form reusado respeta la estructura
original (no degrada a express ni infla filas vacias).

### Snapshots everywhere

`Calculation` guarda TODOS los valores como snapshot
(`printerNameSnapshot`, `printerWattsSnapshot`, `pricePerBobbinSnapshot`,
`gramsPerBobbinSnapshot`). Borrar un filamento del catalogo NO afecta
cotizaciones historicas. Soft FK a printers (nullable + snapshot) para
que borrar la impresora mantenga el nombre y watts historicos.

### Dashboard autoDispose

`dashboardStatsProvider` se re-corre cuando `calculationsNotifierProvider`
emite nuevo state (despues de save/delete/toggle sold). AutoDispose
libera memoria al salir de Home.

### List page: no widget tests

Pagina trivial (lista + PopupMenu + nav). Se confia en la lectura del
notifier (testeado) y el provider chain. Sin tests per user pref
("los tests seran lo minimo").
