## Context

3dCal shippeo Material 3 "Industrial" theme (commit `b9fed6d`) y un `AppScaffold` adaptativo (commit `81f923e`). El branch `refactor/frontend-polish-v2` consolido tokens (`AppSpacing`, `AppRadii`, `AppTheme.defaultStar`), extrajo 8 shared widgets (`StatTile`, `SectionCard`, `MoneyRow`, `SectionHeader`, `ConfirmDialog`, `DefaultBadge`, `AvatarIcon`, `MaxWidthScrollView`, `NumericInputField`), migro strings a `EsBO` y agrego `Semantics` a listas.

Lo que quedo **pendiente o debil** y motiva este PRD:

- **Theme M3 es ad-hoc**: paleta custom hardcoded en `app_theme.dart` (8 colores `Color(0xFF...)`), no se usa `ColorScheme.fromSeed` completo ni el M3 `surface` tonal system. Hay `_lightSurfaceBg` y `_lightCardBg` que duplican roles del `ColorScheme`.
- **Tipografia es la default de Flutter** (`Typography.blackMountainView`) con overrides de `fontWeight`. Sin Google Fonts, sin personalidad, sin jerarquia `display/headline/title/body/label` consistente.
- **Tokens existen pero ~40 sitios siguen con literales** (`EdgeInsets.symmetric(horizontal: 16)`, `BorderRadius.circular(16)`, `fontSize: 14` magicos en `calculations_list_page`, `calculator_page`).
- **A11y es superficial**: hay `Semantics` en listas pero faltan focus rings, skip links, semantic labels en chart, announcements de error, validacion accesible en forms. Touch targets <48dp en chips, icons-only buttons, `IconButton` con `size: 16-20`.
- **Dark mode "carbon"** se ve OK pero usa `surfaceContainerLow/Highest` a ojo, sin auditar contraste WCAG 2.2.
- **Responsive web**: `MaxWidthScrollView` se aplica en 7 sitios pero `AppScaffold` y `home_page` siguen con breakpoints fijos (500, 600, 1024, 1280) sin mediaquery tokens.

**Quien tiene el problema**: Marcelo (mantenedor) quiere que cambiar tokens se propague, no que cada feature tenga su propio `EdgeInsets`. Usuarios web/desktop quieren densidad de informacion mayor. Usuarios mobile quieren touch targets reales. Operador con luz baja quiere dark mode que no canse.

**Por que ahora**: el primer pase dejo la base limpia. Este PRD es la **segunda ola** que sube el liston visual + cierra a11y + aplica tokens masivamente. Sin esto, cada feature nueva re-introduce drift.

## Goal

Re-trabajar la **base del design system** de 3dCal para que **todas las pantallas existentes mejoren solas** (consistencia, legibilidad, accesibilidad, densidad) **sin agregar features de producto** y **sin cambiar comportamiento funcional**.

**Failure consequence** si no se hace:
- Cada nueva feature reinventa `EdgeInsets` y `BorderRadius` → drift permanente.
- 3dCal queda visualmente "decente" pero no profesional (tipografia generica, contraste justo, a11y incompleto).
- Web desktop queda con layout mobile-escalado, no aprovecha el ancho.
- Touch targets <48dp en mobile genera frustration y errores (operador cotiza mal).
- Auditorias WCAG futuras (si se quiere publicar) encuentran gaps caros de cerrar tarde.

## In Scope

1. **Theme M3 nativo**: reescribir `AppTheme.light()` / `AppTheme.dark()` usando `ColorScheme.fromSeed` con seed calibrado + ajustes tonales M3 (`surfaceContainer`, `surfaceContainerLow`, etc.) sin overrides literales innecesarios.
2. **Tipografia con Google Fonts** (si Q1 resuelve afirmativo) o **TextTheme M3 completa** con jerarquia `display/headline/title/body/label` y `letterSpacing` por nivel. Cubre `tabularFigures()` en valores monetarios.
3. **Spacing token compliance**: migrar todos los `EdgeInsets.symmetric(all: N)` / `SizedBox(height: N)` / `Padding` con N literal a `AppSpacing.*`. Auditoria grep + reemplazo sistematico.
4. **Radii token compliance**: migrar `BorderRadius.circular(N)` con N literal a `AppRadii.*`. Idem.
5. **Shared widgets v2**: revisar los 14 widgets existentes, refinar APIs (parametros faltantes: `semanticLabel`, `onTap` ripple consistente, `dense` flag), y agregar los que faltan (ver `### Widgets to add`).
6. **Dark mode parity**: ambos modos pasan auditoria de contraste WCAG 2.2 AA en `body`, `label`, `title`, `error`, `tertiary`. Sin literales `Colors.white24` ni alpha magicos.
7. **A11y WCAG 2.2 AA**:
   - Touch targets ≥48dp en todo lo tappable.
   - Contraste minimo 4.5:1 (texto normal), 3:1 (texto grande/graficos).
   - `Semantics` labels en chart, errores de form, decorative icons (con `ExcludeSemantics`).
   - Focus visible en web (keyboard navigation).
   - Error announcements via `Semantics` live regions.
8. **Responsive mobile/web**:
   - Breakpoints como constantes en `AppBreakpoints` (`mobile < 600`, `tablet 600-1024`, `desktop 1024+`, `wide >= 1280`).
   - Web puede tener maximos de ancho mas generosos (1024 para detail pages, 1280 para dashboards con chart).
   - Mobile compact, desktop mas denso (mas contenido por viewport).

## Out of Scope

- **Nuevas features de producto**: no se agrega catalogo de nozzles, no se agrega export PDF, no se agrega multi-moneda. **Es refactor visual/estructural, no funcional.**
- **Cambios al motor de calculo** (`CalculationEngine`): no se toca logica de pricing.
- **Cambios al modelo de datos** (drift tables, repositories, DAOs): schema intacto.
- **Internacionalizacion adicional**: sigue es_BO hardcoded, BOB unica moneda. Migracion completa a `intl`/`ARB` queda para otro PRD.
- **Sync / cloud / backend**: sigue 100% offline. Esto pesa especialmente si Q1 resuelve agregar Google Fonts (debe descargar en build time, no runtime CDN).
- **Animaciones nuevas** mas alla de las que ya existen (shimmer skeleton, `AnimatedSize` en calculator). No se agrega `motion` choreography.
- **Refactor de `go_router`** o de la estructura de navegacion. Solo consume el theme.
- **Material Symbols** (reemplazar `Icons.x` por `material_symbols_icons`). No en este PRD.
- **Step 12 del polish v2** (unify list tile pattern): si se retoma, va como sub-tarea explicita, no absorbido aca.

## Blocking Questions

> **No escribir plan hasta resolver.** Cada respuesta cambia el alcance materialmente.

### Q1 — Tipografia: Google Fonts vs system fonts

- **Opcion A**: agregar `google_fonts: ^6.2.1` con **2 fuentes** (display + body, ej. `Inter` + `Manrope` o `Plus Jakarta Sans` + `Inter`). Bundle crece ~150-300KB en web (subset por idioma es_BO). En mobile, assets se compilan con la app.
  - Pro: personalidad, jerarquia clara, mas "producto".
  - Contra: dep nueva, decision de cuales 2, riesgo de flicker en web si no se precarga.
- **Opcion B**: mantener `Typography.material2021()` base + solo ajustar `fontWeight`/`letterSpacing` por nivel (lo que hay hoy, refinado).
  - Pro: cero deps, offline por definicion, bundle igual.
  - Contra: tipografia Roboto default = "generic app".

**Recomendacion**: A, con `Inter` (UI) + `JetBrains Mono` (solo para valores numericos/BOB, no como body). Justifica el PRD completo.

### Q2 — Seed color de la paleta

- **A**: mantener `0xFF1B4D7A` (azul tecnico) como seed. Recalibrar `tertiary` y `surfaceContainer*` derivados de M3.
- **B**: explorar seed alternativo (ej. teal industrial `0xFF006A6A`, slate `0xFF334155`).

**Recomendacion**: A. Cambio de seed es estetica pura y se puede iterar despues.

### Q3 — A11y scope

- **A**: **WCAG 2.2 AA estricto**: contraste + touch targets + focus visible + Semantics + error announcements + skip links en web. Scope ~3-5x vs B.
- **B**: **solo lo minimo**: touch targets 48dp + Semantics en controles + contraste auditado en colores principales. Sin focus rings ni skip links.

**Recomendacion**: A, pero aceptar B como MVP si user prioriza velocidad.

### V1 — Hero monetary totals (calculator + calculation detail)

**Motivacion**: la cifra principal del calculator (resultado) y el "Total" del detail son el outcome primario de cada pantalla. Estaban en `displaySmall` (36sp) y `headlineSmall` (24sp), leian como parrafos en vez de como hero.

**Cambios aplicados**:
- `lib/features/calculation/presentation/pages/calculator_page.dart` (linea 1274-1287): el "Big price" del result card sube de `displaySmall` (36sp) a `displayMedium` (45sp). Wrap en `FittedBox(fit: BoxFit.scaleDown)` para que numeros largos (`BOB 1,234,567.89`) no rompan el layout en mobile angosto (320dp). Bold + JetBrains Mono + tabular preservados.
- `lib/features/calculation/presentation/pages/calculation_detail_page.dart` (linea 326-351): el "Total" del detalle sube de `headlineSmall` (24sp) a `headlineMedium` (28sp). Wrap en `FittedBox` por la misma razon. El label companion `Total` sube de `titleMedium` a `titleLarge` para acompanar el peso del valor.

**Commit**: `2697434 feat(calc): promote primary monetary totals to hero display style`

### V2 — Hero EmptyView (lib/shared/widgets/empty_view.dart)

**Motivacion**: empty states eran 80dp circles con `titleMedium` y `bodyMedium`. Leián como alertas chiquitas, no como estado primario de la pantalla.

**Cambios aplicados**:

**Commit**: `e97edfb feat(widgets): make EmptyView hero-shaped with bigger icon and titleLarge`

### V3 — SectionHeader con background tonal (lib/shared/widgets/section_header.dart)

**Motivacion**: section headers eran una fila plana sobre el background de la pagina. Se perdian entre cards.
