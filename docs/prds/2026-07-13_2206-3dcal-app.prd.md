## 1. Vision

App **mobile + web** que permite a operadores de impresion 3D cotizar trabajos en menos de 30 segundos, manteniendo un catalogo local de filamentos/impresoras, historial persistente y dashboard de rentabilidad real del taller.

**Motivacion**: el flujo manual (planilla + calculadora + memoria) genera errores de pricing y no permite ver si el negocio es rentable. 3dcal cierra ese loop con datos del propio operador.

## 5. Non-Functional Requirements

### NFR-1: Performance
- Recalculo de cotizacion: < 100ms con hasta 10 materiales.
- Animacion de expansion Avanzado: < 250ms.
- Apertura de app (cold start mobile): < 2s.
- Build size web: < 3MB initial (target, canary).

### NFR-2: Precision numerica
- **PROHIBIDO `double` en motor de calculo**. Paquete `decimal` o `int` centavos.
- Redondeo final a 2 decimales con `Decimal.toStringAsFixed(2)`.
- Tests unitarios con casos limite: 0.1+0.2, valores muy grandes, descuentos al 100%.

### NFR-3: Reliability
- Persistencia: 0% perdida en cierre de app / reload de web.
- Crash recovery: al reabrir, last state del formulario se restaura (draft local en memoria).
- Validacion defensiva: no se permite guardar cotizacion con peso=0 o tiempo=0.

### NFR-4: Privacy
- Cero telemetria. Cero requests de red. Cero analytics.
- Manifest web no debe cargar CDNs externas (fonts/icons pueden ser locales o no).

### NFR-5: Test coverage
- Unit tests del CalculationEngine: 100% de branches.
- Widget tests de pantallas criticas: Home (Express + Avanzado), History, Dashboard.
- Integration test del flow "crear cotizacion -> guardar -> marcar vendida -> ver dashboard".

### NFR-6: Localizacion
- es_BO unico idioma MVP. Strings en archivo unico `lib/l10n/es_BO.dart` (no usar `intl` con ARB por simplicidad).
- Moneda: `BOB` symbol + format con separador de miles `.` y decimales `,` (convención BO).

### NFR-7: Build & deploy
- Web: `flutter build web --release` produce build estatico deployable a cualquier hosting.
- Mobile: `flutter build apk --release` (Android) y `flutter build ios --release` (iOS, requiere Mac).
- CI opcional: GitHub Actions con build web automatico.

### 6.3 Motor de calculo (pure Dart)

```dart
// lib/features/calculation/domain/calculation_engine.dart
// SIN imports de Flutter. Testeable con `dart test`.

class CalculationEngine {
  static CalculationOutput compute({
    required List<MaterialInput> materials,
    required Decimal printerWatts,
    required Decimal totalHours,
    required Decimal discountPercentage,
    required Decimal profitBasePercentage, // 200% default
    required Decimal kwhRate,              // 0.70 BOB/kWh default
  }) {
    // material cost
    final materialCost = materials.fold<Decimal>(Decimal.zero, (acc, m) {
      final pricePerGram = m.pricePerBobbin / m.gramsPerBobbin;
      return acc + (m.weightGrams * pricePerGram);
    });

    // electric cost
    final electricCost = totalHours * (printerWatts / Decimal.fromInt(1000)) * kwhRate;

    final baseCost = materialCost + electricCost;
    final effectiveProfit = profitBasePercentage - (discountPercentage * Decimal.fromInt(2));
    final profitAmount = baseCost * (effectiveProfit / Decimal.fromInt(100));
    final totalPrice = baseCost + profitAmount;

    return CalculationOutput(
      baseCost: baseCost,
      profitAmount: profitAmount,
      effectiveProfitPercentage: effectiveProfit,
      totalPrice: totalPrice,
      materialCost: materialCost,
      electricCost: electricCost,
    );
  }
}
```

## 7. Calculation Engine — Reglas formales

### 7.1 Inputs validos

| Variable | Tipo | Rango | Default |
|----------|------|-------|---------|
| `materials[i].weightGrams` | Decimal | > 0 | n/a |
| `materials[i].pricePerBobbin` | Decimal | > 0 | n/a |
| `materials[i].gramsPerBobbin` | Decimal | > 0 | n/a |
| `totalHours` | Decimal | > 0 | n/a |
| `printerWatts` | Decimal | >= 0 | 0 (sin impresora) |
| `kwhRate` | Decimal | 0.10 - 5.00 | 0.70 |
| `profitBasePercentage` | Decimal | 0 - 1000 | 200 |
| `discountPercentage` | Decimal | 0 - 50 | 0 |

### 7.2 Formulas

```
materialCost = Σ(weightGrams[i] × pricePerBobbin[i] / gramsPerBobbin[i])
electricCost = totalHours × (printerWatts / 1000) × kwhRate
baseCost = materialCost + electricCost
effectiveProfit = profitBasePercentage - (discountPercentage × 2)
profitAmount = baseCost × (effectiveProfit / 100)
totalPrice = baseCost + profitAmount
```

### 7.3 Edge cases

- **effectiveProfit < 0**: si descuento > 50% con profitBase < 100%, el resultado es negativo. Decision MVP: **clampear a 0** y mostrar warning en UI. NO rechazar el calculo.
- **weightGrams = 0**: boton guardar disabled.
- **printerWatts = 0**: electricCost = 0, se omite del breakdown.
- **Redondeo**: todas las salidas se redondean a 2 decimales al final con `Decimal.toStringAsFixed(2)`. NO redondeos intermedios.

| R4 | Scope creep (filtros, export, sync) | Alta | Alto | Out-of-scope explicito en este PRD. Tickets separados. |
