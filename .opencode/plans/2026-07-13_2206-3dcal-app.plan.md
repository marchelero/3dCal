## 4. Riesgos tecnicos y mitigaciones (foco implementacion)

| # | Riesgo | Mitigacion en implementacion |
|---|--------|------------------------------|
| R1 | drift web performance pobre | Sprint 0: elegir `drift_flutter` (combina `sqlite3_flutter_libs` mobile + `sqlite3.wasm` web con IndexedDB). Sprint 2 bench: query 100 calculations en < 50ms. Si falla, migrar a `package:drift/web/worker.dart` con `WasmDatabase` separado. |
| R2 | Material 3 + AnimatedList jank en low-end | Sprint 4: usar `AnimatedSize` + `AnimatedSwitcher` en vez de `AnimatedList` puro (mas predecible). Sprint 8 profile mode + reduce motion. |
| R3 | Decimal package overhead | Sprint 1: bench con 1000 computations, target < 50ms total. Si > 200ms, fallback a `int` centavos (multiplicar todo por 100, dividir al final). Documentado como decision. |
| R4 | Scope creep | Plan respeta PRD 3.2. Cualquier feature nueva va a "Future" section del CHANGELOG. Si el usuario pide algo fuera de scope durante implementacion, marcar como ticket separado, NO encolar. |
| R5 | Diseno UI sin designer | Material 3 defaults + seed color deep purple (TBD). No custom typography ni custom components en MVP. Whitelabel-friendly. |
| R6 | iOS build requiere Mac | Sprint 9: skip iOS si no hay Mac, documentar en README. Android es target primario. |
| R7 | Tests drift en web no triviales | Sprint 2: tests de repos usan `NativeDatabase.memory()` (funciona en host con sqlite3). NO se intenta testear drift web en CI (limite conocido). |
| R8 | Browser Chrome no instalado | Sprint 0: usar `flutter run -d edge` o instalar Chrome. Sprint 9: `flutter build web` no requiere browser local. |
| R9 | drift `DecimalColumn` requiere sqlite >= 3.24 con extension. Web sqlite3.wasm puede no traer | Sprint 0: validar version wasm >= 0.5 con extension decimal. Si no, almacenar precios como `int` centavos (multiplicar por 100) y formatear al mostrar. Decision lock en Sprint 1. |
| R10 | Codegen drift + riverpod tarda > 30s en incremental | Sprint 2: usar `--build-filter` para codegen incremental por archivo. Si sigue lento, dividir `app_database.dart` por dominio (calculation_db, catalog_db, settings_db) — pero intentar evitar por complejidad. |

## Decision log (cambios respecto al PRD)

1. **Database location**: PRD 6.2 muestra `drift_database.dart` en `lib/features/calculation/data/`. Movido a `lib/core/database/app_database.dart` porque la DB es compartida por TODAS las features (filaments, printers, calculations, settings). Cohesion > seguir literalmente el path del PRD.
2. **Filaments/Printers tables**: extraidas de "calculation" en PRD 6.4 a sus propias features `lib/features/catalog/filaments/data/filaments_table.dart` y `lib/features/catalog/printers/data/printers_table.dart`. Coherente con feature-first.
3. **AC-3 expected value**: PRD dice "Bs. 43.04" pero calculo exacto es Bs. 42.98. El test del Sprint 1 validara la formula matematica, no el numero del PRD. Se documenta en code comment del test.
4. **Form builder**: PRD menciona "flutter_form_builder o custom" sin lockear. Este plan usa **custom** (TextFormField + Riverpod notifier) para control fino + integracion con Decimal. Justificacion: validation custom para Decimal > 0, evitar dependencia extra.
5. **Draft recovery**: PRD open question 12.2 dice "decision MVP: NO". Este plan **incluye draft recovery en Sprint 8** porque NFR-3 (Reliability) menciona "Crash recovery: al reabrir, last state del formulario se restaura". Esos dos se contradicen en el PRD; el plan prioriza NFR-3. Si el usuario quiere estrictamente NO draft, eliminar Sprint 8F.
6. **Chrome no instalado**: pre-flight nota que `flutter run -d chrome` fallara. Plan usa `flutter run -d edge` como alternativa. Build web (`flutter build web`) no requiere browser local. No bloqueante.
7. **iOS build**: PRD 5.7 (NFR-7) menciona iOS release. Plan marca iOS como "skip si no hay Mac" en Sprint 9. Out-of-scope practico para Windows-only dev.
8. **CI**: PRD 5.7 marca "opcional". Plan incluye `.github/workflows/ci.yml` solo si el usuario lo pide explicitamente. Default: NO.
9. **Snapshot deletion test**: PRD AC-14 dice "Eliminar filamento PETG usado en cotizacion historica → cotizacion mantiene snapshot".
